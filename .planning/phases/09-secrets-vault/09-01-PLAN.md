<objective>
Build the cryptographic foundation and on-disk storage layout for the per-project secrets vault. Output of this plan is a fully unit-tested `secrets/` package that can encrypt/decrypt secrets to/from `vault.tqv` blobs using a per-project master key resolved from OS keyring or env-passphrase, without yet being wired into MCP tool surface or the migration runner.
</objective>

<tasks>

1. Add `keyring>=24`, `cryptography>=42`, `argon2-cffi>=23` to `pyproject.toml` runtime deps. Run `uv sync` to refresh lockfile. Verify imports succeed.

2. Create module `src/turbo_memory_mcp/secrets/crypto.py`:
   - `encrypt(plaintext: bytes, key: bytes) -> bytes` ŌĆö AES-256-GCM, prepends 12-byte random nonce to ciphertext + 16-byte GCM tag.
   - `decrypt(blob: bytes, key: bytes) -> bytes` ŌĆö inverse; raises `InvalidTag` from `cryptography` if MAC fails.
   - `derive_key_from_passphrase(passphrase: str, project_id: str) -> bytes` ŌĆö Argon2id with `time_cost=3`, `memory_cost=64*1024` KiB, `parallelism=4`, salt = `sha256(b"tqv-salt-v1:" + project_id.encode()).digest()`, output 32 bytes.
   - Constants: `NONCE_SIZE = 12`, `TAG_SIZE = 16`, `KEY_SIZE = 32`.
   - 100% unit coverage: round-trip, wrong-key fails, tampered ciphertext fails, KDF determinism per `(passphrase, project_id)`.

3. Create module `src/turbo_memory_mcp/secrets/keyresolver.py`:
   - Internal `KeyResolutionMode` enum: `ENV`, `KEYRING_EXISTING`, `KEYRING_BOOTSTRAPPED`. Not exported via `__init__.py` ŌĆö internal detail.
   - `resolve_master_key(project_id: str) -> tuple[bytes, KeyResolutionMode]`:
     1. If env var `TQMEMORY_SECRETS_PASSPHRASE` is set -> `derive_key_from_passphrase(env_value, project_id)` -> return `(key, ENV)`. Env wins as explicit user intent (precedence locked in 09-CONTEXT.md).
     2. Try `keyring.get_password("turbo-quant-memory", f"secrets-master-{project_id}")`. If present, base64-decode to 32 bytes -> return `(key, KEYRING_EXISTING)`.
     3. If absent but `keyring.get_keyring()` resolves to a writable non-fail backend: generate 32 random bytes via `secrets.token_bytes(32)`, base64-encode, store via `keyring.set_password(...)`, return `(key, KEYRING_BOOTSTRAPPED)`.
     4. Otherwise: raise `MasterKeyUnavailable` with a multi-line setup message containing the exact `keyring set ...` and `export TQMEMORY_SECRETS_PASSPHRASE=...` commands plus a one-line explanation of why interactive prompt is intentionally not used (reboot survival).
   - Detect keyring writability with a short try/except on a sentinel `keyring.set_password("turbo-quant-memory", "__probe__", "1")` followed by `delete_password` ŌĆö done lazily, only when bootstrap is the candidate path. Probe failure -> path (4).
   - Unit tests use a fake in-memory backend that supports both "writable" and "fail" modes to cover all four branches without touching the real OS keychain.

4. Create module `src/turbo_memory_mcp/secrets/store.py`:
   - `class SecretsStore` constructed with `(storage_root: Path, project_id: str)`.
   - Paths: `secrets_dir = storage_root / "projects" / project_id / "secrets"`, `vault_path = secrets_dir / "vault.tqv"`, `meta_path = secrets_dir / "meta.json"`.
   - `provision()` ŌĆö idempotent. Creates `secrets_dir` with mode `0o700`, writes empty `vault.tqv` (encrypted JSON `{"version": 1, "entries": {}}`) and `meta.json` (`{"version": 1, "kdf": "argon2id", "kdf_params": {...}, "key_mode": "<keyring|env>", "created_at": "<iso>"}`) if missing. Files get mode `0o600`. Master key is resolved here via `keyresolver`.
   - `set(name: str, value: str) -> None` ŌĆö load+decrypt vault, set entry with `updated_at`, re-encrypt+atomic-write.
   - `get(name: str) -> str | None` ŌĆö load+decrypt, return entry value or None.
   - `list_names() -> list[str]` ŌĆö load+decrypt, return sorted entry names. NEVER values.
   - `delete(name: str) -> bool` ŌĆö remove entry if present, re-encrypt+write; return whether existed.
   - Atomic write: `tempfile.NamedTemporaryFile` in same dir + `os.replace`.
   - Name validation: `^[A-Za-z0-9_.-]{1,128}$`, reject otherwise.
   - Unit tests: provision idempotency, set/get/list/delete round-trip, persist across new instance, tampered vault file raises clearly.

5. Create module `src/turbo_memory_mcp/secrets/audit.py`:
   - `class AuditLog` constructed with `(secrets_dir: Path)` ŌĆö **per-project**, lives inside the same `secrets/` directory as `vault.tqv`. Project deletion removes audit too; matches project-scope-only invariant.
   - `record(action: str, name: str) -> None` ŌĆö append a JSON line to `secrets_dir / "audit.jsonl"`. Schema: `{"ts": "<iso-utc>", "action": "set|get|list|delete", "name": "..."}`. `project_id` is implicit from path; value is never recorded.
   - File created with mode `0o600`, parent dir already `0o700` from `SecretsStore.provision()`. Append-only via `os.open(path, O_WRONLY | O_APPEND | O_CREAT, 0o600)`; one fsync per record.
   - Unit tests: schema correctness, no value in any field, append semantics, parent-dir reuse with `SecretsStore`, file permission check.

6. Create `src/turbo_memory_mcp/secrets/__init__.py` exporting the public surface (`SecretsStore`, `AuditLog`, `MasterKeyUnavailable`, `resolve_master_key`).

</tasks>
